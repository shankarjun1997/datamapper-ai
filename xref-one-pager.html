xREF DataMapper

Agentic source-to-target mapping for enterprise data-warehouse & cloud migrations
Who it's for: data & migration teams modernizing warehouses / moving on-prem → cloud, cloud → cloud, or consolidating databases — architects, migration leads, analysts, and governance / change-advisory boards.
! The problem
  • Thousands of tables/columns to map — manual source→target mapping is slow and error-prone
  • No way to know if a migration is safe before writing ETL — type mismatches surface late
  • No lineage or impact visibility; changes break downstream silently
  • Weak governance: who approved what, and can it be audited for sign-off?
✦ What it delivers
  • Confidence-scored mappings (exact · fuzzy · semantic) with human review gates
  • A migration-readiness verdict per column, table, and program — before ETL
  • End-to-end lineage + impact analysis from one canonical metadata model
  • Audit-ready governance & sign-off artifacts for the change board
→ Input · Processing · Output
📥 Input
Source schemas (CSV / DDL / Excel, multi-file) or live crawl of Oracle, SQL Server, Postgres, MySQL, Snowflake, BigQuery, Redshift, Databricks. Optional target schema + business context (Jira / instructions).
→
⚙ Processing
4-stage agentic pipeline: L1 schema intelligence → L2 target schema → L3 semantic mapping + confidence → L4 spec. Plus a readiness rules engine, lineage/impact, and a versioned metadata repository.
→
📤 Output
Mapping spec (STM CSV — column & table level), Mapping Report (HTML / XLSX), Migration Readiness Certificate (PDF), lineage graph, and a Change-Management Bundle (ZIP) for the CAB.
🛡 Governance
  • Role-based access (admin · mapper · reviewer · readonly)
  • Human approval gates — low-confidence mappings cannot be auto-approved
  • Versioned mappings + multi-tenant data isolation
  • Secrets encrypted at rest; SSO & audit-log (SIEM) export
⚖ Accountability
  • Every action audited — who, what, when, from where
  • Sign-off workflow: Analyst → Architect → Lead
  • Readiness Certificate carries a SHA-256 verification hash
  • AI assists; a human owns the approval — full traceability
